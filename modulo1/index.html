O que são Clients?

Cliente é quando você consome algo de algum lugar. Nesse caso, client vai ser o dispositivo que está consumindo conteúdo (informação) de um site hospedado na internet. Hospedar um site é colocar o seu arquivo de internet (arquivo do seu site) em algum servidor o qual pode retornar arquivos estáticos (a partir de sua solicitação):

.HTML
.CSS
.JavaScript
.Imagens
.etc

O que são Servers?

Servidores são onde ficam armazenadas as informações. Eles não existem apenas na internet. Podem ser servidores locais. Tendem a ser um computadore mais robusto por necessitar de mais processamento devido às várias ligações que os mesmos têm que gerenciar. Tipos de servidores:

.arquivos: computador onde ficará centralizado as imagens, os áudios, os vídeos, que todo mundo pode acessar.
.segurança (firewall).
.streaming: dedicados a realizar o streaming de dados. Fornece dados à medida em que os usuários vão solicitando.
.email: vai armazenar e gerenciar os emails enviados e recebidos.
.web: fornece arquivos da internet (HTML, CSS, Javascript, imagens, vídeos, etc).

Ainda importante mencionar que datacenter é o local onde ficam armazenados vários servidores

Como é uma estrutura básica de um HTML?

HTML

Não é uma linguagem de programação e sim um arquivo de marcação. São textos delimitados por nomes que o navegador consegue interpretar. Cada nome destes delimitadores tem uma função e um comportamento específico (semântica). A mudança do HTML 4 para o HTML 5 trouxe um foco maior na semântica. Tira-se a função de formatação e fortalece a de estruturação (formatação -> css) Semântica é a:
Disciplina da linguística que se ocupa da significação das palavras e expressões linguísticas bem como das relações de sentido que estas estabelecem entre si.

Mudanças do HTML 5

Estrutura básica

Header:cabeçalho
Main: possui o conteúto principal da página. Tag principal.

.Não pode (deve) ser filha de header ou footer
.Só deve haver 1 (uma) por documento HTML
.Não deverão incluir seções da página que sejam comuns a todo o site ou aplicação, tais como mecanismos de navegação (utilizar nav), informações de copyright (footer), logotipo (header) ou campos de busca (utilizar nav. A não ser quer "Buscar" seja o objetivo principal do site. Ex.: google)

Footer: informações de rodapé da página.

.adress: tag de endereço. Por padrão, o texto fica em itálico

Outras tags

Section: representa uma seção genérica contida em um documento HTML, geralmente com um título, quando não existir um elemento semântico mais específico para representá-lo. Por exemplo, um menu de navegação deve estar dentro de um elemento nav, mas uma lista de resultados de pesquisa pode ser colocada numa section.

Aside: assuntos que não têm ligação direta (tangencialmente relacionados) com o conteúdo da página.

Nav: representa uma seção de uma página que aponta para outras páginas ou para outras áreas da página, ou seja, uma seção com links de navegação.

Article

.time pubdate: tag semântica que especifica de quando é o artigo.
.article: é possivel criar um dentro do outro. O buscador entenderá que se tratam de "pai" e "filho" (artigos correlacionados).

Blockquote: tag para grande citação. Por padrão, desloca do resto do texto e aplica um recúo.
.cite="": especifica a fonte da citação (um link para um site, por exempo)

q: tag para pequena citação. Por padrão, acrescenta aspas ao texto nela contido.

.cite="": especifica a fonte da citação (um link para um site, por exempo)

Figure: funciona como um container que delimita o escopo de uma imagem (tag img). Tudo que estiver dentro se relaciona entre sim no âmbito de uma imagem.

.figcaption: tag auxiliar que funciona como uma descrição para a imagem. É uma legenda ou descrição da imagem. É pública para todos verem (diferentemente do atributo "alt" em img).

Picture: é possível utilizar múltiplas fontes para a mesma imagem.

.source src="": tag para indicar a fonte da imagem.
.media="(min-width: 600px)": atributo que indica que, se a tela tiver até 600px, será utilizada essa fonte (mobile first - media query css)
